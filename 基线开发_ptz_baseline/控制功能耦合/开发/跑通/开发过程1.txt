四、解决方案：

下面给你一套“从根上解耦普通 camera 与 ptz camera”的改法。  
核心思想：**不要再靠工具名字符串前缀分发 provider**，统一改为 **`device-mappings` 的业务归属分发**（`getIntentForTool`）。

## 现存问题
1. 当前分发逻辑仍包含 `toolName.startsWith("camera")`，`camera_zoom` 会被误分到普通 camera provider。  
2. 分发逻辑在两个地方重复，后续改一处漏一处，持续引入冲突风险。

```ts
    return this.executeToolCalls(state, "find_target_tool");
  }

  private async getToolsByOwnership(
    toolName: string,
    userId: string,
    language: ChatbotSupportedLanguage,
  ): Promise<StructuredToolInterface[]> {
    const intentType = getIntentForTool(toolName);

    if (intentType === "camera") {
      return this.cameraToolProvider.generateCameraTools(
        userId,
        toPrismaLanguage(language),
      );
    }

    if (intentType === "ptz_camera") {
      return this.ptzToolProvider.generatePtzTools(
        userId,
        toPrismaLanguage(language),
      );
    }

    if (intentType === "unknown") {
      this.logger.warn(
        `[ToolDispatch] Unknown tool ownership: ${toolName}, fallback to control provider`,
      );
    }

    return this.toolProvider.generateControlTools(userId);
  }

  /**
   * 通用工具执行
   */

```

更改后解决了什么问题：
- 把“普通 camera / PTZ camera”分发规则集中到一个方法，彻底摆脱名称前缀耦合。  
- 后续新增 PTZ 或 camera 工具时，只要更新 `device-mappings`，无需再改多处分发 `if-else`。
